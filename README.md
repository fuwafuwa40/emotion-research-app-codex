# EmotionLab - 感情相関分析 研究用プロトタイプ

EmotionLabは、発話に含まれる感情表現、本人の自己報告感情、そしてウェアラブル端末から得られる生体データの関係を分析するための研究用モバイルアプリケーションです。診断や治療を目的とした医療機器ではなく、感情と身体反応の相関を探索するための研究用MVPとして設計されています。

## 研究目的

本プロトタイプは、以下の3種類のデータ間の関連性を探索することを目的としています。

| データ種別 | 内容 | 取得方法 |
|-----------|------|---------|
| 発話感情データ | テキストから抽出した感情極性スコア | 簡易辞書ベースの感情分析 |
| 自己報告感情データ | Valence（快・不快）とArousal（覚醒度） | 9段階のSAMスケール |
| 生体データ | 心拍数（HR）と皮膚コンダクタンス（SC） | 手動入力（将来的にウェアラブル連携） |

## システム構成

本アプリはローカルファーストのアーキテクチャを採用しており、全データは端末内のAsyncStorageに保存されます。サーバーやクラウドへのデータ送信は行いません。

## ローカル起動

最短でWebプレビューを開く場合は、依存関係を入れた上で次を実行します。

```bash
pnpm install
pnpm web
```

`pnpm web` は空いているポートを自動で選び、Expo Web を起動してブラウザで開きます。

バックエンドも含めて開発する場合は次です。

```bash
pnpm dev
```

Vercel にデプロイする場合は、静的 Web 出力を使います。リポジトリ側の設定は `vercel.json` に入っているので、Vercel 側で別の build command を手で設定する必要はありません。

```bash
pnpm build:web
```

### 技術スタック

本プロトタイプは以下の技術で構成されています。React Native (Expo SDK 54) をベースとし、TypeScriptによる型安全な開発を行っています。UIスタイリングにはNativeWind (Tailwind CSS) を使用し、状態管理はReact Context + AsyncStorageで実現しています。

### コアモジュール

| モジュール | ファイル | 役割 |
|-----------|---------|------|
| 型定義 | `lib/types.ts` | Session, SentimentResult, CorrelationResult等のデータモデル |
| 感情分析 | `lib/sentiment.ts` | 日英対応の辞書ベース感情極性分析（否定語処理含む） |
| 統計分析 | `lib/statistics.ts` | ピアソン相関係数、記述統計、正規化 |
| ダミーデータ | `lib/dummy-data.ts` | 20件のリアルなダミーセッション生成 |
| セッション管理 | `lib/session-store.tsx` | AsyncStorage永続化 + React Context Provider |

## 画面構成

### Sessions タブ（ホーム画面）

セッション一覧を時系列で表示します。各カードには発話テキストの冒頭、感情ラベル（Positive/Neutral/Negative）、自己報告値（V/A）、生体データ（HR/SC）が表示されます。「+20 Dummy」ボタンでダミーデータを一括生成でき、実データがなくても動作確認が可能です。

### Record タブ（データ入力画面）

3ステップのウィザード形式でデータを入力します。Step 1では発話テキストを入力し、Step 2ではValence（1-9: 非常に不快〜非常に快）とArousal（1-9: 非常に低覚醒〜非常に高覚醒）を選択します。Step 3では心拍数と皮膚コンダクタンスを入力します（ダミー値生成ボタンあり）。

### Analysis タブ（分析ダッシュボード）

収集されたデータの統計分析結果を表示します。画面上部にはセッション数、平均Valence、平均心拍数のサマリーカードが並びます。中段には各変数の時系列推移を正規化バーチャートで表示し、下段にはピアソン相関係数の一覧と記述統計テーブルが表示されます。

### Session Detail 画面

個別セッションの全データを詳細表示します。発話テキスト全文、感情分析結果（ラベルとスコア）、自己報告データ（Valence/Arousal）、生体データ（HR/SC）を確認できます。

## 分析手法

### 感情分析（辞書ベース）

日本語と英語の感情語辞書（ポジティブ58語、ネガティブ55語）を用いた極性分析を行います。否定語（「ない」「くない」「ません」「not」等）の検出による極性反転にも対応しています。スコアは -1.0（完全にネガティブ）〜 +1.0（完全にポジティブ）の範囲で算出されます。

### 相関分析

ピアソンの積率相関係数（r）を用いて、以下の変数ペア間の線形相関を算出します。

| 変数ペア | 分析の意図 |
|---------|-----------|
| AI感情スコア × Valence | 発話感情と主観的感情の一致度 |
| AI感情スコア × Arousal | 発話感情と覚醒度の関係 |
| AI感情スコア × 心拍数 | 発話感情と生理的反応の関係 |
| Valence × 心拍数 | 主観的感情と生理的反応の関係 |
| Valence × 皮膚コンダクタンス | 主観的感情と自律神経反応の関係 |
| Arousal × 心拍数 | 覚醒度と心拍の関係 |
| Arousal × 皮膚コンダクタンス | 覚醒度と自律神経反応の関係 |
| 心拍数 × 皮膚コンダクタンス | 生体指標間の関係 |

相関係数の解釈は |r| >= 0.7 を「強い相関」、0.4 <= |r| < 0.7 を「中程度の相関」、|r| < 0.4 を「弱い相関」としています。

## 設計判断の根拠

### ローカルファーストを選択した理由

研究用プロトタイプの初期段階では、サーバー構築やデータベース管理のオーバーヘッドを避け、迅速にプロトタイピングを進めることが重要です。AsyncStorageによるローカル保存は、個人研究者が単一端末で実験する場合に十分であり、プライバシーの観点からも安全です。

### 辞書ベース感情分析を選択した理由

機械学習モデル（BERT等）を使用する選択肢もありましたが、以下の理由から辞書ベースを採用しました。説明可能性が高く、なぜその判定になったかを研究者が理解できること。小規模データでも安定して動作すること。外部APIへの依存がなく、オフラインで動作すること。将来的にモデルベースの分析を追加する際の比較ベースラインとして機能すること。

### SAMスケール（9段階）を選択した理由

Self-Assessment Manikin (SAM) は感情研究で広く使用されている標準的な尺度です。Valence（快・不快）とArousal（覚醒度）の2次元モデルは、Russell (1980) の感情円環モデルに基づいており、研究の再現性と比較可能性を確保できます。

## ダミーデータ

「+20 Dummy」ボタンで生成されるダミーデータは、以下の特徴を持つように設計されています。日本語の自然な発話テキスト（ポジティブ・ニュートラル・ネガティブ）が含まれ、Valence/Arousalは発話内容と一定の相関を持つよう調整されています。心拍数は60-100bpmの生理的に妥当な範囲で生成され、皮膚コンダクタンスは1.0-5.0uSの範囲で生成されます。

## テスト

ユニットテストはVitestで実装されており、以下のテストスイートが含まれます。

```bash
cd /home/ubuntu/emotion-research-app
npx vitest run
```

| テストファイル | テスト数 | 内容 |
|--------------|---------|------|
| `lib/__tests__/sentiment.test.ts` | 4 | 感情分析（ポジティブ/ネガティブ/ニュートラル/否定語） |
| `lib/__tests__/statistics.test.ts` | 6 | 統計計算（平均/SD/相関/正規化） |

## 将来の拡張方針

本プロトタイプは以下の方向に拡張可能な設計となっています。ウェアラブルデバイス（Apple Watch, Fitbit等）からのリアルタイム生体データ取得、音声認識による発話テキストの自動入力、機械学習ベースの感情分析モデルの追加、時系列分析（ラグ相関、グレンジャー因果性）の実装、複数参加者のデータ管理とグループ分析、そしてCSV/JSONエクスポート機能の追加が想定されています。
